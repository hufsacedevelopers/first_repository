# 프론트엔드 개선 포인트

## 목적
이 문서는 현재 프론트엔드 구현 상태를 기준으로, 다음 작업에서 우선적으로 개선할 항목을 정리한 문서입니다.  
디자인 고도화, 실제 API 연동, 배포 전략 논의 시 참고용으로 사용합니다.

## 현재 상태 요약
- Next.js + TypeScript + Tailwind + App Router 기반으로 초기 구조가 구성되어 있음
- 메인 페이지 디자인은 목업 기준으로 구현되어 있음
- 기업 리스트, 추천 일자리, 지원금 섹션이 화면에 표시됨
- 기본 화면은 mock 데이터 중심으로 동작
- 일부 백엔드 live API는 구현되었지만 프론트는 아직 직접 연결하지 않음

## 우선 개선 항목

### 1. 데이터 소스 전환 방식 정리
- 현재는 `NEXT_PUBLIC_API_URL` 존재 여부에 따라 mock / 실제 API를 전환함
- 배포 환경에서 예기치 않게 live API를 바라보며 오류가 날 수 있음
- 개선 방향:
  - 개발/배포 환경별 데이터 전략 명확화
  - mock 모드 / live 모드를 명시적으로 제어할 수 있도록 개선
  - 에러 발생 시 fallback UI 제공

### 2. API 에러 처리 UI 추가
- 현재 API 응답이 실패하면 페이지 전체가 에러로 깨질 수 있음
- 개선 방향:
  - 섹션별 로딩/에러/빈 데이터 상태 컴포넌트 추가
  - `try/catch` 기반 서버 데이터 로딩 보강
  - 사용자에게 “현재는 mock 데이터로 표시 중” 같은 안내 제공

### 3. live API와 프론트 데이터 매핑 정리
- 백엔드에는 live API가 추가되었지만 프론트 타입은 주로 mock 구조에 맞춰져 있음
- 개선 방향:
  - `/jobs/live`, `/companies/standard-workplaces/live`, `/supports/incentives/live` 응답을 프론트 타입으로 매핑
  - 서버 응답용 타입과 화면 표시용 타입을 분리
  - formatter 함수(`mapper`, `transformer`) 도입

### 4. 기업 점수 카드 고도화
- 현재 점수 카드는 시각적으로는 구성되어 있으나 실제 계산 근거와 연결되지는 않음
- 개선 방향:
  - 세부 지표 설명 툴팁 또는 설명 텍스트 추가
  - 표준사업장 인증 여부, 지원금 가능 여부 등 live 정보 반영
  - 점수 산식이 붙으면 카드와 방법론 영역 연결

### 5. 추천 일자리 카드 실제 데이터 연결
- 현재 추천 일자리 카드는 mock 데이터 기반
- 개선 방향:
  - `jobs/live` 기반으로 실제 공공데이터 표시
  - `jobs/live-with-env` 기반으로 작업환경 조건 태그 반영
  - 추후 AI 추천 점수 필드가 들어갈 자리 미리 확보

### 6. 접근성 및 공공서비스 UI 보강
- 현재 디자인은 깔끔하지만 공공서비스 성격에 맞는 접근성 점검이 더 필요함
- 개선 방향:
  - 명도 대비 점검
  - 키보드 포커스 스타일 보강
  - 시맨틱 태그 및 스크린리더 친화 문구 보강
  - 버튼/링크 역할 구분 명확화

### 7. 배포 전략 확정 후 환경변수 정리
- 프론트 배포 플랫폼(Vercel / Railway / 보류)이 아직 확정되지 않음
- 개선 방향:
  - 팀 논의 후 프론트 배포 플랫폼 확정
  - `NEXT_PUBLIC_API_URL` 운영값 정리
  - 도메인 기준 CORS 정책과 함께 문서화

## 추천 작업 순서
1. API 에러 처리와 fallback UI 추가
2. live API 응답을 프론트 타입으로 매핑하는 레이어 추가
3. 추천 일자리/기업 카드에 실제 API 연결
4. 기업 점수 계산 로직 연결
5. 접근성 개선
6. 배포 환경 정리

## 팀 논의가 필요한 항목
- 프론트 배포를 지금 바로 진행할지 여부
- 프론트 배포 플랫폼을 `Vercel`로 할지 `Railway`로 할지
- MVP 단계에서 mock 우선인지, live API 우선인지
- AI 추천 기능을 이번 단계에 일부라도 보여줄지 여부

## 메모
- 현재 백엔드는 Railway 배포 기준으로 거의 준비된 상태
- 프론트는 배포보다 먼저 “실패하지 않는 데이터 연결 방식”을 정리하는 것이 우선
- 문서 업데이트 후 실제 구현은 작은 단위로 나눠서 진행하는 것이 좋음
